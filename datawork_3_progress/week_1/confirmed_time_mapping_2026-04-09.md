# 시간 키 매핑 규칙 확인 메모

## 결론

`tariff_hour = 기준시 - 1` 규칙을 프로젝트의 확정 규칙으로 채택한다.

즉,

- `기준시 = 1` → `hour = 0` → `00:00 ~ 01:00`
- `기준시 = 24` → `hour = 23` → `23:00 ~ 24:00`

## 왜 확정할 수 있는가

이번 규칙은 외부 규정과 공개 설명을 기준으로 정리했다.

### 근거 1. 전력시장운영규칙

- 전력시장운영규칙은 각 거래시간이 해당 단위기간의 `끝점`으로 표시된다고 정의한다.
- 예시로 `거래시간 06시`는 `05:00 직후부터 06:00에 종료하는 기간`이라고 설명한다.
- 또한 거래일은 `거래시간 01시`에서 시작해 `거래시간 24시`에 종료한다고 정의한다.

이 규칙을 시간대별 전력 데이터에 적용하면:

- `1시`는 `00:00 ~ 01:00`
- `24시`는 `23:00 ~ 24:00`

으로 해석하는 것이 자연스럽다.

### 근거 2. 공공데이터포털 전력시장 데이터 설명

- 공공데이터포털의 전력시장 데이터 설명도 각 시간이 해당 단위기간의 `끝점으로 표시`된다고 설명한다.
- 따라서 전력시장/전력수요/전력거래 계열 시간 데이터는 `1~24` 끝점 표기를 공통 규칙으로 사용하는 것으로 볼 수 있다.

## 내부 데이터와의 정합성

외부 근거뿐 아니라 현재 프로젝트 데이터 구조와도 모순이 없다.

- 실제 부하 원본의 `기준시` 범위는 `1~24`
- `기준시 = 24`가 실제로 존재한다.
- `tariff_tou.csv`의 `hour` 범위는 `0~23`
- 평균 부하 패턴도 `기준시 - 1` 해석과 구조적으로 충돌하지 않았다.

## 프로젝트 적용 규칙

앞으로 `지역본부별 시간대별 전력사용량` 데이터를 `tariff_tou.csv`와 연결할 때는 아래 규칙을 사용한다.

1. `기준일자`에서 `month` 추출
2. `기준시`를 정수형으로 읽기
3. `hour = 기준시 - 1`
4. `month`, `hour` 기준으로 `tariff_tou.csv`와 조인
